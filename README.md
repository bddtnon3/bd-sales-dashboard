# BD Sales Dashboard — เซิร์ฟเวอร์กลาง (Vercel + Blob)

เว็บรายงานยอดขายทีมเซลล์ BD Distribution แบบ **อัพครั้งเดียว ทุกคนเห็นพร้อมกัน**
แอดมินอัพไฟล์ยอดขาย (รายวัน / รายเดือน+เป้า / รายร้าน) ผ่านหน้าเว็บ → บันทึกขึ้นส่วนกลาง → เซลล์ทุกคนเปิดดูได้

---

## โครงสร้างไฟล์

```
bd-sales-dashboard/
├─ public/
│  └─ index.html        ← หน้าเว็บ (กราฟ, drill-down ร้าน, วิเคราะห์ — ฝังไลบรารีครบ)
├─ api/
│  ├─ login.js          ← ตรวจรหัสผ่าน คืน token
│  ├─ data.js           ← ส่งข้อมูลล่าสุดให้หน้าเว็บ (ต้องมี token)
│  ├─ save.js           ← แอดมินอัพข้อมูล → เก็บลง Blob (เฉพาะ manager)
│  └─ seed-data.json    ← ข้อมูลเริ่มต้น ม.ค.–ก.ค. 2026 (ใช้จนกว่าจะอัพครั้งแรก)
├─ lib/
│  ├─ auth.js           ← ระบบ token (เซ็นชื่อกันปลอม)
│  └─ accounts.js       ← รายชื่อผู้ใช้ + รหัสผ่าน (อยู่ฝั่งเซิร์ฟเวอร์ ไม่หลุดถึงเบราว์เซอร์)
├─ package.json
└─ .gitignore
```

## ขั้นตอน Deploy (ทำครั้งเดียว ~10 นาที)

### 1) เอาโค้ดขึ้น GitHub
สร้าง repo ใหม่ (เช่น `bd-sales-dashboard`) แล้ว push โฟลเดอร์นี้ทั้งหมดขึ้นไป
```bash
cd bd-sales-dashboard
git init
git add .
git commit -m "BD Sales Dashboard - central server"
git branch -M main
git remote add origin https://github.com/<ชื่อคุณ>/bd-sales-dashboard.git
git push -u origin main
```

### 2) สร้างโปรเจกต์ใน Vercel
- เข้า Vercel → **Add New… → Project** → เลือก repo `bd-sales-dashboard` → **Import**
- Framework Preset ปล่อยเป็น **Other** (ไม่ต้องตั้ง build command) → **Deploy**

### 3) สร้าง Blob store (ที่เก็บข้อมูลกลาง)
- ในโปรเจกต์ Vercel → แท็บ **Storage → Create Database → Blob** → ตั้งชื่ออะไรก็ได้ → **Create**
- กด **Connect** ให้ผูกกับโปรเจกต์นี้ → Vercel จะเพิ่มตัวแปร `BLOB_READ_WRITE_TOKEN` ให้อัตโนมัติ

### 4) ตั้งค่า Environment Variables
ไปที่ **Settings → Environment Variables** เพิ่ม:

| ชื่อ | ค่า | จำเป็น |
|---|---|---|
| `AUTH_SECRET` | ข้อความสุ่มยาวๆ (เช่นจากคำสั่ง `openssl rand -hex 32`) | **ใช่** — กันปลอม token |
| `MANAGER_PASS` | รหัสผ่านผู้จัดการที่คุณอยากตั้ง (เช่น `BDadmin#2026`) | แนะนำ |
| `PASS_CT11` … `PASS_KA98` | ตั้งรหัสรายคน (ถ้าไม่ตั้ง จะใช้ค่า default เช่น `ct11@bd`) | ไม่บังคับ |

> ตั้ง env แล้วกด **Redeploy** หนึ่งครั้งเพื่อให้มีผล

### 5) ใช้งาน
- เปิด URL ที่ Vercel ให้ (เช่น `https://bd-sales-dashboard.vercel.app`)
- Login เป็น **manager** → กด "อัพเดตข้อมูล" → อัพไฟล์ 3 ชนิด (รายวัน / รายเดือน+เป้า / รายร้าน)
  ระบบจะเซฟขึ้นส่วนกลาง — เซลล์ทุกคนรีเฟรชแล้วเห็นทันที
- แจก URL + รหัสให้ทีมเซลล์

> มีข้อมูล ม.ค.–ก.ค. 2026 ฝังไว้เป็นค่าเริ่มต้นแล้ว เปิดปุ๊บเห็นเลยก่อนอัพครั้งแรก

## รหัสผู้ใช้เริ่มต้น (เปลี่ยนได้ผ่าน Environment Variables)

| ผู้ใช้ | รหัสผ่าน (default) | สิทธิ์ |
|---|---|---|
| `manager` | `bd@admin` (หรือค่า `MANAGER_PASS`) | เห็นทุกสาย + อัพโหลดข้อมูล |
| `ct11` … `ct17`, `ct22` | `<ชื่อผู้ใช้>@bd` เช่น `ct11@bd` | เซลล์ (เห็นสายตัวเอง + ภาพรวมทีม) |
| `ka97`, `ka98` | `ka97@bd`, `ka98@bd` | เซลล์ |

**สำคัญ:** เปลี่ยนรหัสก่อนใช้งานจริงผ่าน env `PASS_CT11` ฯลฯ และตั้ง `AUTH_SECRET` เสมอ

## การอัพเดตข้อมูลประจำ (แอดมิน)
| ไฟล์ | ความถี่ | ปุ่ม |
|---|---|---|
| ยอดขายรายวัน (รายการบิล) | ทุกวัน | 📅 รายวัน |
| ยอดรายเดือน + เป้าหมาย | เดือนละครั้ง | 🎯 รายเดือน+เป้า |
| ยอดรายร้าน (by store `.xls`) | เดือนละครั้ง (หรือรายสัปดาห์) | 🏪 รายร้าน |

อัพผ่านหน้าเว็บได้เลย ไม่ต้องแตะโค้ดหรือ Vercel อีก

## หมายเหตุทางเทคนิค
- ไม่ต้องมี build step — เป็น static + serverless functions ล้วน
- ข้อมูลเก็บเป็นไฟล์ JSON ก้อนเดียวใน Vercel Blob (เขียนทับทุกครั้งที่อัพ เก็บเฉพาะชุดล่าสุด)
- ไลบรารีกราฟ/อ่าน Excel ฝังในหน้าเว็บ → ใช้ได้แม้เครือข่ายบล็อก CDN ภายนอก
- ต้องการรันทดสอบในเครื่อง: `npm i -g vercel && vercel dev`
